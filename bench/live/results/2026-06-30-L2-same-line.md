# Live run — L2 same-line collision — 2026-06-30

- **Rung:** L2 (two agents change the same line, conflicting intent)
- **Agents:** 2 general-purpose coding agents (Claude), real edits, real decisions
- **Seed repo:** `pool.js` with `maxConnections()` returning `100`.
- **Scenario:** two tickets land at once, both targeting the same function:
  - `perf-agent` (PERF-412): raise `maxConnections()` to **500** — peak load is
    dropping requests at the ceiling.
  - `safety-agent` (SAFETY-87): drop `maxConnections()` to **25** — the DB gets
    overwhelmed during incidents.

  These are directly opposed on the same line. `perf-agent` edits first and is
  still in-flight (uncommitted, as fleet agents are between coordination points)
  when `safety-agent` arrives.
- **Conditions:** WITHOUT Quilt (plain git) vs WITH Quilt (claim before editing).

## Result: WITHOUT silently loses a fix; WITH catches it before any damage

| | WITHOUT Quilt | WITH Quilt |
|---|---|---|
| perf's `500` after both agents run | **gone** (0 commits, not in tree) | **preserved** in the working tree |
| signal that a conflict happened | **none** | claim denied + `fleet` shows it |
| safety's action | overwrote perf blindly, committed | **held off**, escalated for a decision |
| final state | `25`, perf's work silently lost | nothing lost, conflict surfaced |

## WITHOUT Quilt — a fix vanished and nobody was told

`perf-agent` changed `maxConnections()` to `500` and left it uncommitted (a fleet
agent mid-task). `safety-agent`'s edit tool read the file — saw `500` — and
changed it to `25`, then committed. Plain git raised no conflict: a sequential
commit over an uncommitted change is not a merge conflict, it's just an
overwrite.

The damage, measured directly:

```
final value:        return 25;
perf's 500 in git:  0 commits   (git log -S 500 → nothing)
working tree:       500 not present
stash / reflog:     nothing
```

PERF-412's fix exists nowhere. Both agents reported success; the perf agent
believes production can now handle peak load. It can't — its change was
overwritten before it was ever committed, with no signal to anyone.

## WITH Quilt — the collision was refused, not absorbed

`perf-agent` claimed the symbol before editing:

```
✓ claimed pool.js#maxConnections
```

When `safety-agent` tried to claim the same symbol, Quilt **denied** it:

```
✗ denied  pool.js#maxConnections (held by perf-agent)
```

and `quilt fleet` showed exactly why:

```
Quilt · fleet   2 actors, 0 clashes, 1 blocked
  Actors
    ● perf-agent (agent)   reserved, not yet edited
        claims: pool.js#maxConnections
  Blocked
    ⛔ safety-agent waiting on pool.js#maxConnections (held by perf-agent)
```

`safety-agent` did not edit. From its own report:

> "A denied claim means another agent is mid-change on that code, so I'm
> reporting rather than forcing the change through. Beyond the mechanical
> collision, PERF-412 (500) and SAFETY-87 (25) are directly opposed on the same
> value — this needs a human/coordination decision, not two agents overwriting
> each other's number."

perf's `500` stayed in the working tree, intact. Nothing was committed silently.
The conflict — both the mechanical one and the genuine disagreement of intent —
was put in front of a human instead of being resolved by whoever happened to
write last.

## What this proves

- **The silent-overwrite failure is real with real agents.** WITHOUT Quilt, one
  agent's uncommitted fix was overwritten and lost with no conflict, no warning,
  and two "successful" agents. This is the L1 follow-up case (a harder rung than
  disjoint) made concrete.
- **Advisory claims stop it before it happens.** The protection here came from
  the claim denial, which is live the instant a symbol is reserved — it does not
  wait for attribution or commit. The second agent was told the code was taken
  and chose to coordinate.
- **The signal turns a silent collision into a deliberate decision.** Quilt's job
  wasn't to pick 25 or 500 — it was to make sure a human gets to. That's the
  confidence: agents can fan out on the same checkout and a real clash surfaces
  instead of disappearing.

(Companion to the L1 disjoint run — where the work didn't truly collide and both
conditions happened to succeed — and the L3 cascade run, where push-awareness
caught a cross-file break. L2 is the same-line case the earlier runs flagged as
the missing rung.)
