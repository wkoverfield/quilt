Measured, not promised

The same six scenarios, with and without Quilt.

Run 2026-07-12 · Quilt 0.4.6 · reproduce with npm run bench · raw results in the repo

Quilt ships with a graded scenario ladder of multi-agent failure modes, drawn from live two-agent experiments. Every scenario runs twice on the same tasks and the same shared working tree: once with plain git (no isolation, no coordination) and once with Quilt (actors claim before editing and commit with commit --mine). Both runs are graded on the same metrics, and the WITH-Quilt assertions gate every CI run, so these numbers can't silently drift from what the code actually does.

The headline

Across the ladder (18 intended changes, 2 to 4 concurrent actors):

across all six scenarioswithout Quiltwith Quilt
changes silently lost30

Features landed is identical across conditions on every rung: where the baseline silently dropped a change, Quilt surfaced the same collision instead. The difference is what happens to the losing edit: without Quilt it vanishes with no signal; with Quilt it becomes a decision.

How to read this honestly

  • The scripted layer encodes cooperative agents. Actors claim before editing and back off on denial. It proves what the coordination contract delivers when followed, not that every agent follows it; a live sub-agent layer tests real agents against the same ladder.
  • Harness wall clock is not latency. Each scripted action spawns a fresh CLI process, so the harness measures process startup. Real per-edit capture is free (about 0.01 ms) and reconcile stays under about 70 ms even at 150 changed files; see LATENCY.md.
  • The no-redo baseline is deliberate. When plain git silently loses an edit, nobody redoes it, because nobody knows. That is the failure being measured.
  • Capture coverage is measured too. Across 13 real Claude agents on bash-tempting tasks, 100% of file writes went through capturable tools; the one theoretical gap (raw shell edits) fails safe: surfaced as unattributed, never miscredited. See the capture eval.
