Quickstart

$ npm install -g @quilt-dev/cli
$ cd your-repo
$ quilt setup     # wires capture hooks + the optional MCP server
$ quilt doctor    # confirms it's wired and capture is flowing

quilt setup detects your orchestrator and wires the repo: capture hooks, the shared MCP server in .mcp.json, and a coordination snippet in CLAUDE.md. It is idempotent, and --dry-run shows what would change. Commit the generated config so every checkout shares the wiring; .quilt/ itself stays local and gitignored.

Four terminals, one repo

$ quilt setup      # once
$ claude           # terminal 1
$ claude           # terminal 2, 3, 4...

Nothing else. Each Claude Code session or MCP connection is named automatically, every edit is attributed to its session as it happens, and a write into code another session has claimed is denied before any bytes change. Want a stable, readable id? QUILT_ACTOR=auth-agent claude.

Watch it live and commit when ready:

$ quilt ui                        # the dashboard, local-only
$ quilt status                    # who owns what
$ quilt commit --mine -m "fix auth redirect"
  ✓ Committed 1 file(s) (b41e7a2). Other actors' changes remain in the working tree.
Claude Code note: the optional MCP claim tools appear after a restart plus a one-time /mcp approval. The capture hooks protect every edit either way, and quilt claim / quilt commit --mine always work from the shell.

Commands

A global --as <id> sets your actor for any command. QUILT_ACTOR wins over --as.

commandwhat it does
quilt setupWire the repo (or a whole workspace of repos): hooks, MCP server, coordination snippet. Idempotent; --dry-run previews.
quilt doctorHealth check: wired, identity set, capture flowing, git version, MCP server live-test.
quilt uiThe fleet dashboard in your browser: who wrote what, claims, blocked and queued actors, the needs-you queue. Binds 127.0.0.1 only.
quilt fleetThe same view in the terminal; --watch refreshes live, --json for programs.
quilt statusWho owns which working-tree changes.
quilt claimReserve files, directories, or symbols (file#symbol) before editing, with --intent "why". A denial names the holder, their intent, and when their lease lapses. --queue auto-grants when the target frees.
quilt releaseRelease your claims (all of them if no paths given).
quilt commit --mineCommit exactly your lines as an ordinary Git commit, even where files are shared. Auto-releases your claims on the committed files. preview --mine shows the patch first.
quilt escalateFlag a collision agents can't reconcile for a human; shows under "needs you".
quilt resolveMark a collision handled; --take --from <actor> transfers their dirty operations to you.
quilt undo / restoreBack out one actor's uncommitted changes, or recover work another actor overwrote (both versions are preserved on detection).
quilt watchAttribute edits live on file events and catch collisions the moment they happen.
quilt telemetryShow or change anonymous usage telemetry. Off by default, strictly opt-in.
quilt mcpRun the MCP server (stdio) for agent integration.

Full flags: quilt <command> --help, or the complete reference.

How attribution works

One governing rule: a blocked commit beats a wrong one. Uncaptured work is surfaced as unattributed, never guessed into the wrong author's commit.

  • Capture at the tool boundary (primary). Hooks and the MCP edit tools record each edit's author the moment it happens, into an append-only ledger. Any line the ledger knows wins.
  • Claims (prevention and binding). A claim reserves both editing and attribution. Edits that bypass capture (bash, codegen) bind to whoever claimed the file first.
  • Reconcile inference (the floor). Each command diffs the tree against the last observed snapshot and attributes the delta to the actor running it, gated so a contested tree is never absorbed by whoever reconciles first.

commit --mine keeps only the lines you own, applies that patch to a throwaway temporary index, and produces an ordinary Git commit. Your real index and working tree are never rewritten, and other actors' changes stay exactly where they were. Attribution keys each change by side, symbol scope, content, and occurrence, so identical lines in different functions can belong to different actors.

Running your agents

Two ways to capture. On Claude Code, hooks capture every native edit with zero ceremony. Everywhere else, the shared MCP server (quilt mcp) exposes the same capture and prevention as tools: start_session, claim, get_conflicts, commit_mine, quilt_edit, and friends.

  • Claude Code: first-class (hooks + MCP). Subagents get their own ids automatically.
  • Codex CLI: capture via ~/.codex/hooks.json, wired by setup. Requires a one-time interactive hook approval; capture-only today.
  • Cursor: via MCP (.cursor/mcp.json, wired when a .cursor/ directory exists).
  • Anything else: route edits through quilt_edit / quilt_write, or claim first and edit however you like.

The loop that pays: claim before editing (whole files normally, file#symbol to share a file), don't force a denial (queue, adapt, or escalate), and finish with commit --mine. Several subagents sharing one process? Give each a stable id and pass it as actor on every call.

State and safety

Everything lives in a local .quilt/ sidecar: actors, sessions, per-line ownership, the append-only capture ledger, live claims, preserved overwrites, and the escalation queue. Delete it any time; your repo is untouched.

  • Quilt never calls an LLM, never spawns agents, never rewrites Git history. Every commit it produces is an ordinary Git commit.
  • Claims are advisory coordination for cooperative agents, not a sandbox. The sharp edges, stated plainly.
  • POSIX only for now: macOS and Linux.
  • Telemetry is off by default and strictly opt-in; when enabled it sends anonymous counts only, never code, paths, or names.
