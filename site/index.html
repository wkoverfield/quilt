The argument

Worktrees isolate.
They don't coordinate.

A worktree per agent is the reflex answer, and for fully independent tasks it works. But isolation has two costs that grow with every agent you add.

The bill. Every worktree is another environment: another install, another build cache, another dev server to keep alive. The whole point of a shared checkout is paying for the environment once.

The blindfold. Isolated agents can't see each other's in-flight work, so the collision isn't prevented, it's deferred to merge time, where it's biggest and you're the one holding it. Agents that could have adapted to each other mid-task each build against a world that no longer exists.

The two aren't mutually exclusive: worktrees for independent, long-running work, Quilt for agents in the same code at once.

How it works

Attribution at the tool boundary,
ownership per line.

01 · capture

Every edit is signed as it happens

Capture hooks record each edit's author at the tool boundary. Each Claude Code session or MCP connection is named automatically: no ceremony, no protocol for the agent to follow, nothing to approve.

02 · coordinate

Claims make intent visible

Agents reserve files, directories, or single symbols (utils.js#formatPrice, ten languages via tree-sitter). A write into claimed code is denied before any bytes change, and the denial carries the holder's stated intent and when their lease lapses.

03 · commit

Each actor commits only its lines

quilt commit --mine reconstructs exactly your patch, even where two actors share a hunk, and produces an ordinary Git commit. Everyone else's uncommitted work stays untouched, and your claims auto-release.

$ QUILT_ACTOR=builder-flows quilt claim deals.js flows.js \
    --intent "wire flows to deals"
  ✗ denied  deals.js (held by builder-friction)
      builder-friction is: friction pass: rename + archive flags
      their claim lapses 22:12:05Z unless renewed
  ✓ claimed flows.js

$ quilt commit --mine -m "wire flows to deals"
  ✓ Committed 1 file(s) as builder-flows (03d9c11).
    Other actors' changes remain in the working tree.

A denied claim isn't a dead end: the blocked agent builds its granted files, re-claims after the holder's commit auto-releases, and layers on top. Nothing lost.

Built in

When agents collide, nothing is lost.

Push-awareness

Claim a symbol that depends on a function another actor is changing, and Quilt warns you at claim time, so the dependent adapts instead of building against a stale signature.

$ quilt claim billing.js#total
  ✓ claimed billing.js#total
  ⚠ heads-up: total depends on rate,
    which codex is changing (billing.js#rate)

Detect and preserve

If one actor overwrites another's uncommitted lines anyway, Quilt snapshots the victim's version and says so. Both versions survive; you choose.

⚠ collision  claude-ui overwrote codex's
  edits in auth.ts. both saved, run:
$ quilt restore auth.ts

Straight answers

Small tool, sharp edges stated.

  • All state lives in a local .quilt/ sidecar. Delete it any time; your repo is untouched.
  • Quilt never calls an LLM, never spawns agents, never rewrites Git history.
  • Claude Code is first-class (hooks + MCP). Codex is capture-only today; Cursor and Aider via MCP tools.
  • POSIX only for now: macOS and Linux, no Windows/CRLF.
  • Claims are advisory coordination for cooperative agents, not a sandbox.
  • Bash and codegen edits aren't auto-captured: claim first, and they bind to you. Uncaptured edits are surfaced, never miscredited.
