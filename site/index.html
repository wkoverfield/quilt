One repo. Every agent. Clean commits.

Quilt tracks which agent wrote which lines in a shared checkout, so all your agents work in one repo, against one dev server, and each one commits exactly its own changes.

01 · capture

Every edit is attributed

Hooks record each edit's author as it happens. Agents are named automatically; nothing to call, nothing to approve.

02 · coordinate

Claims prevent collisions

A write through your agent's edit tools into claimed code is denied before any bytes change. Beyond those tools claims are advisory: claim first and the edit binds to you.

03 · commit

commit --mine

Each agent commits exactly its own lines as an ordinary Git commit. Everyone else's work stays untouched.

Worktrees isolate. They don't coordinate.

Every worktree is another install, build, and dev server, and its agent is blind to everyone else's in-flight work, so collisions just wait for merge time. Quilt keeps the whole fleet on one checkout, accounting for each other as they go.
