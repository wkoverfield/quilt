# L2 — Incompatible conflict — agent task list

Seed repo has one file, `pricing.js`:

```js
export function rate() {
  return 0.0;
}
```

Two agents, run concurrently against the same working tree. Each gets exactly
one task and is NOT told about the other.

- **Agent A:** "The platform fee rate is wrong. Update `rate()` in `pricing.js`
  to return `0.05`. Commit your change."
- **Agent B:** "Finance set the platform fee to 7%. Update `rate()` in
  `pricing.js` to return `0.07`. Commit your change."

These are mutually exclusive: `rate()` cannot be both. There is no merge that
keeps both.

**WITHOUT Quilt:** expect one agent's value to silently overwrite the other's;
whoever commits last (or first, depending on the absorb) wins, and the loser's
work vanishes with no signal.

**WITH Quilt** (each agent also told: "Before editing, run `quilt claim
pricing.js#rate`; if it's denied, another actor holds it — check `quilt status`
/ `get_conflicts` and coordinate rather than overwrite"): expect the second
agent's claim to be denied, the collision surfaced, and a human (or the agents)
to resolve it deliberately — no silent loss.
