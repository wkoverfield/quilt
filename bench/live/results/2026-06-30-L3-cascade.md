# Live run — L3 dependency cascade — 2026-06-30

- **Rung:** L3 (dependency cascade)
- **Agents:** 2 general-purpose coding agents (Claude)
- **Seed repo:** `src/money.js` exports `cents(dollars)`; `src/invoice.js` imports
  it and calls it inside `lineItem(price)`.
- **Scenario:** one agent changes a function another depends on.
  - `money-agent`: rename `cents` → `toCents` in `money.js` (a breaking change for
    every caller).
  - `invoice-agent`: unrelated task — add an `amount` field to `lineItem` (which
    calls the renamed function).
- **Conditions:** WITHOUT Quilt (plain git) vs WITH Quilt (claim → adapt → commit).
  The refactor lands first in both, then the caller agent does its task.

## Result: the codebase BROKE without Quilt, and RAN with it

Each repo's `invoice.js` was imported and `lineItem(4.25)` was actually executed:

| | WITHOUT Quilt | WITH Quilt |
|---|---|---|
| `money.js` exports | `toCents` | `toCents` |
| `invoice.js` imports/calls | `cents` (stale) | `toCents` (fixed) |
| **does it run?** | **✗ broken** | **✓ works** |

```
WITHOUT → ✗ SyntaxError: './money.js' does not provide an export named 'cents'
WITH    → ✓ { amountCents: 425, amount: 4.25 }
```

## WITHOUT Quilt — the cascade broke (and blindness, even with the clue)

`invoice-agent` did its task (added the `amount` field) and committed. It left
the import and call as `cents`, which `money.js` no longer exports. The module no
longer imports — the build is broken.

The striking part: the agent *noticed* the rename and still did nothing. From its
own report:

> "the commit log shows a prior commit `rename cents to toCents` (likely from the
> other agent), but `invoice.js` still imports `cents` — the import name was not
> affected by my change."

With no signal that *demanded* action, a focused agent stayed in its lane and
shipped broken code. That's exactly the silent cascade.

## WITH Quilt — push-awareness caught it, the agent fixed it

`invoice-agent` claimed `src/invoice.js#lineItem` and got a heads-up at claim time:

```
✓ claimed src/invoice.js#lineItem
⚠ heads-up src/invoice.js#lineItem depends on cents, which money-agent is changing (src/money.js#cents)
```

It investigated (read `money.js`), found the rename, fixed the import **and** the
call site to `toCents`, then did its own task and committed. From its report:

> "The Quilt heads-up worked as intended: it surfaced money-agent's rename so I
> could reconcile invoice.js against the current money.js rather than committing
> code broken by a stale import."

The fix came from the agent itself — Quilt's job was to *direct its attention* to
the dependency in flux. Same task, same agent capability; the only difference was
the signal.

## What this proves

- **The dependency-cascade win is real with real agents.** WITHOUT, the dependent
  shipped broken; WITH, push-awareness turned a silent cascade into a fixed one,
  and the agent never had to remember to check — the claim told it.
- **Push-awareness works end to end in practice**, not just in the unit tests:
  the heads-up fired on a real cross-file reference and a real agent acted on it
  correctly.
- This is the live counterpart to the **L3** rung in the scripted ladder
  (`bench/`), which proves the same failure/fix deterministically.

(Contrast with the L1 disjoint run on the same day: there, both conditions
happened to succeed because the work didn't truly collide. The cascade is where
coordination stops being optional.)
